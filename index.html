PaaS for government roadmap [DRAFT]

Overview

We want to build the PaaS for government based on what you need. This roadmap is a rough indication of what we're planning on building in the near-, medium-, and long-term.

Items on the roadmap are our current estimate when things will be available on the platform for use by users. If you would like more information about anything we mention below please get in touch at gov-uk-paas-support@digital.cabinet-office.gov.uk

Current Offering

The following features are available on the PaaS now. You can read more about the detail in our development documentation.

Platform

  • Deploy apps in numerous languages and frameworks: Python, JVM based, Node.js, Ruby, Go
  • Rapidly Scale App Instances
  • Cross data-center deployment
  • Log shipping to Software as a Service for Apps
  • SSH Access to Containers

Security

  • Official - No Sensitive or Personal Information

Backing Services

  • Postgres

Requested Features

Below are features which people have been asking about, but are not currently in our plans.

Docker

Cloud Foundry supports pushing a Docker image as an app. This feature is not currently enabled on the Government PaaS because allowing deployment from Docker images, where the root filesystem is controlled by the tenant, raises additional security concerns: see this note from the CF developers for more details. We hope to enable support for Docker images in the future. If you would like to be able to push Docker images, please contact our support team at gov-uk-paas-support@digital.cabinet-office.gov.uk, providing details of your use case.

Cloud Foundry Route Services

Cloud Foundry has a feature called route services. This allows you to filter and transform requests before they reach application instances. At present we don't know of any use cases for this feature amongst our users. If you would like to use it, please contact our support team at gov-uk-paas-support@digital.cabinet-office.gov.uk, providing details of what you would like to use it for.

Roadmap

This roadmap is for informational purposes only, so you shouldn't rely on this information for major purchasing or planning purposes. As with all things we may be affected by changing priorities or delays.

Near-term

  • Information Assurance for Personal Data
  • Information Assurance for Sensitve Data
  • Redis Backing Service

Medium-term

  • Elasticsearch as a Backing Service
  • Self Service SSL Cert Management
  • MongoDB as a Backing Service
  • Self Service User Management
  • Improved Backing Service Monitoring

Long-term

  • Private Networking for Apps
  • User Application Monitoring
